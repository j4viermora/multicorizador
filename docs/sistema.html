👥 Actores

  • Super Admin — gestiona todo el sistema
  • Productor — agente de seguros, crea cotizaciones
  • Cliente — viajero, ve cotizaciones vía link público

🏢 Tenants (Companies)

  • Cada Company es un tenant aislado
  • Los productores pertenecen a una company
  • Cada registro scoped lleva company_id
  • La landing pública se accede vía /cotizar/:slug
  • Venta directa de Ruka: existe una Company reservada con slug: "ruka" (Company::RUKA_DIRECT_SLUG) con productores in-house. HomeController#index redirige a los visitantes anónimos de / hacia /cotizar/ruka, reutilizando el mismo modelo tenant/producer que cualquier otra agencia
  • Toda búsqueda en la landing pública crea un Quote + sus QuoteResult — ya no es efímera, incluso las búsquedas anónimas quedan registradas para atribución y comisión

🌐 Integraciones

  • 3 providers fake (AssistCard, UniversalAssistance, TravelAce)
  • Webhooks para confirmar compras
  • Resend en producción (emails)
  • Kamal para deploy con Docker

🏗 Multi-Tenancy

Cómo se aíslan los datos entre companies.

gem acts_as_tenant v1.0.1 — Automáticamente agrega company_id = X a todas las queries y JOINs de los modelos tenant-scoped. El tenant se setea en ApplicationController#set_current_tenant desde current_user.company.

Modelos con acts_as_tenant :company

  • ✓ Quote
  • ✓ QuoteResult
  • ✓ Policy
  • ✓ Traveler
  • ✓ Link

Modelos globales (sin tenant)

  • — Provider
  • — InsurancePlan
  • — Company
  • — User (solo super_admin no tiene company)
⚠️ Los background jobs corren sin contexto HTTP, por lo que no tienen tenant. Deben usar ActsAsTenant.without_tenant para cargar el Quote y luego ActsAsTenant.with_tenant(quote.company) para crear registros.

🔐 Autenticación y Roles

Devise + enums de role y status.

Roles de Usuario

super_admin (1)
Accede a /admin y /jobs. Sin restricción de tenant.
producer (0)
Accede a /producer solo si está active.

Status del Productor

pending (0) Recién registrado, espera aprobación. Redirige a /account/pending.
active (1) Puede usar la plataforma normalmente.
suspended (2) Bloqueado, se cierra sesión automáticamente.

Registro de Productores

Al registrarse via Users::RegistrationsController, se asigna automáticamente role: producer y status: pending. Se crea una Company nueva para el productor. El Super Admin la aprueba desde /admin/users.

M Asociaciones entre Modelos

Relaciones principales entre los modelos ActiveRecord.

Company

  • has_many users (dependent: destroy)

User

  • belongs_to company (optional)
  • has_many quotes (as producer)
  • has_many travelers (as producer)

  • belongs_to company
  • belongs_to producer (User)
  • belongs_to traveler (optional)
  • has_many quote_results (dependent: destroy)
  • has_many links (dependent: destroy)

QuoteResult

  • belongs_to company
  • belongs_to quote
  • belongs_to provider
  • belongs_to insurance_plan (optional)
  • has_one policy (dependent: nullify)

Policy

  • belongs_to company
  • belongs_to quote_result

Provider

  • has_many insurance_plans (dependent: destroy)
  • has_many quote_results (dependent: nullify)

⚙ Background Jobs (Solid Queue)

Jobs asincrónicos gestionados por Solid Queue. En dev corre en proceso separado (Procfile.dev); en producción dentro de Puma (SOLID_QUEUE_IN_PUMA=true).

QuoteJob

Orquesta el proceso de cotización. Actualiza el quote a quoting y encola un ProviderQuoteJob por cada proveedor activo.

Queue:default
Inputquote_id
Tenantwith_tenant(quote.company)

ProviderQuoteJob

Llama al API de un proveedor específico y crea un QuoteResult con el precio devuelto. Cuando todos completan, actualiza el quote a quoted.

Queue:default
Retry3× / 5s backoff
ErrorProviderError
Tenantwith_tenant(quote.company)

WebhookProcessorJob

Procesa webhooks de proveedores. Parsea el payload, crea la Policy y actualiza el quote a purchased.

Queue:default
TriggerPOST /webhooks/:slug

Arquitectura de frontend

Sin Node ni webpack. El CSS se compila con bin/rails tailwindcss:watch. Los imports de JS se manejan con Importmap. Propshaft sirve los assets. Para formularios se usa siempre simple_form_for con f.input — nunca form_with ni helpers crudos de Rails. El I18n default es :es (UI en español, código en inglés).
